From install to first quote.

Short guides for Jira admins and delivery teams setting up Quotaurus in Jira Cloud.

Installation

  1. Open the Quotaurus Atlassian Marketplace listing when available.
  2. Select the Jira Cloud site where the app should be installed.
  3. Review the requested permissions on the Atlassian install screen.
  4. Open a Jira project and look for the project page named Quotes.
  5. Open a Jira issue and look for the issue panel named Quotes.

Installation requires Jira Cloud admin permission. Contact Quotaurus if you need the current Marketplace access route.

Create your first quote

  1. Open a Jira issue or the project Quotes page.
  2. Create a draft quote.
  3. Add client details, title, currency, forecasted completion date, and notes.
  4. Add line items with product, rate, quantity, and VAT setting.
  5. Save the quote, then generate a PDF preview.

Link quotes to Jira issues

Quotaurus can link open quotes to Jira issues. Linked quotes are visible from the issue panel and can be opened back in the project Quotes page.

Generate quote PDFs

Quote PDFs include quote reference and version, project title, quote date, expiry text, prepared-by detail when available, client, PO number, notes, line items, VAT, and totals.

PDF generation does not require Mailgun. Teams can download or share a generated PDF according to their internal process.

Configure Mailgun email sending

Quote email sending uses saved Mailgun project settings. The email includes a short message, quote details, and the generated quote PDF as an attachment. On successful send, Quotaurus stores recipient and sent-date metadata on the quote.

  • Mailgun domain.
  • Mailgun region.
  • Mailgun API key.
  • From email and optional reply-to email.

Recurring quote templates

Recurring templates can generate monthly draft quotes. Templates store client, title, currency, system, revenue stream, PO number, notes, VAT setting, status, line items, and billing day. Schedules support days 1 to 28 and a last-day-of-month option.

Quote statuses

Quotaurus currently uses Draft, Sent, Accepted, Rejected, Invoiceable, Closed, and Cancelled. A quote marked Invoiceable requires a PO number.

Permissions and roles

The current app manifest requests Forge app storage, read Jira work, write Jira work, and read Jira user scopes. Quotaurus does not publish a separate public role model on this site; teams should review Atlassian's install permission screen and their Jira project access model before rollout.

Troubleshooting

  • If the Quotes page is not visible, confirm the app is installed on the correct Jira Cloud site.
  • If an issue panel does not show expected quotes, confirm the quote is open and linked to that issue.
  • If a PDF cannot be generated, include the quote reference and visible error in support email.
  • If email sending fails, check Mailgun domain, region, API key, and from email settings.
  • If a quote cannot be marked Invoiceable, check that a PO number has been added.
