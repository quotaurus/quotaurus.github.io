Set up RevFlow for your Jira workflow.

Short guides for Jira admins and delivery teams configuring RevFlow, the Jira app from Quotaurus for quote-to-delivery revenue control.

Installation

  1. Open the RevFlow Atlassian Marketplace listing or the current Marketplace access route from Quotaurus.
  2. Select the Jira Cloud site where RevFlow should be installed.
  3. Review the requested permissions on the Atlassian install screen.
  4. Open a Jira project and look for the project page named RevFlow.
  5. Open a Jira issue and look for the issue panel named RevFlow.

Installation requires Jira Cloud admin permission. Contact Quotaurus if you need the current Marketplace access route.

First-run setup wizard

When a project is not configured, RevFlow shows project admins a setup banner and opens the first-run setup wizard. The same wizard can be relaunched later from settings.

  1. Welcome and project context.
  2. Jira field mappings.
  3. Quote statuses, work buckets, and issue types.
  4. Company details for quote output.
  5. Default currency, supported currencies, and tax settings.
  6. Quote items, clients, and revenue streams.
  7. Feature flags and optional email provider settings.
  8. Access rules for configuring, creating, editing, sending, deleting, and reporting.
  9. Review, export, import, and finish setup.

Finishing first-run setup starts the guided product tour for the project page workflow.

Jira field mapping

RevFlow can map to Jira fields for systems, acceptance criteria, parent or epic relationships, sprint metadata, estimates, and logged time. The setup wizard uses live Jira field metadata and marks auto-detected suggestions where it can.

Statuses, issue types, and work buckets

RevFlow ships with default quote statuses including Draft, Sent, Accepted, Rejected, Invoiceable, Closed, and Cancelled. Admins can configure quote statuses and map Jira work statuses into reporting buckets such as order, WIP, invoiceable, lost, and closed.

Missing quote coverage reporting uses configured issue types, so teams can focus coverage checks on the Jira work types that matter.

Create and link your first quote

  1. Open a Jira issue or the RevFlow project page.
  2. Create a draft quote and add client, title, currency, PO, forecast completion, notes, system, and revenue stream details.
  3. Add line items with product, rate, quantity, and tax settings.
  4. Save the quote and link it to the Jira issues it covers.
  5. Generate a PDF preview or move the quote through your configured status flow.

Reports

RevFlow includes project reporting views for quote value by status, quote funnel, value over time, top clients, top systems, top revenue streams, billable activity, completion month, and billing exports.

Missing quote coverage reports highlight configured issue types in the current and next sprint that do not have linked quote coverage.

PDFs and optional email sending

Quote PDFs include quote reference and version, project title, quote date, client details, PO number, notes, line items, tax, and totals. PDF generation does not require email sending to be configured.

Email sending is optional. The setup wizard supports provider choices for None, Mailgun, and SendGrid. When a configured provider sends a quote email successfully, RevFlow stores recipient and sent-date metadata on the quote.

Recurring quote templates

Recurring templates can generate monthly draft quotes. Templates store client, title, currency, system, revenue stream, PO number, notes, tax setting, status, line items, and billing day. Schedules support days 1 to 28 and a last-day-of-month option.

Permissions and access

RevFlow configuration includes permission modes for Anyone, Named users, Project admins, and Jira admins. Admins can configure access for setup, quote creation, editing, sending, deletion, and report viewing.

The app manifest requests Forge app storage, read Jira work, write Jira work, and read Jira user scopes. Teams should review the Atlassian install screen before rollout.

Setup import and export

RevFlow setup can be exported and imported as JSON. This is useful for review, repeat configuration, and support conversations about project setup.

Jira dark mode compatibility

RevFlow enables Atlassian Forge theme support and includes dark-mode styles for panels, forms, tables, reports, status pills, overlays, and setup wizard screens when Jira is using dark theme or match-system theme behaviour.

Troubleshooting

  • If RevFlow is not visible, confirm the app is installed on the correct Jira Cloud site.
  • If setup is blocked, confirm the current user has project admin or configured setup permission.
  • If linked quote information is missing, confirm the quote is linked to the relevant Jira issue.
  • If missing coverage looks wrong, review configured issue types, sprint field, and work status mappings.
  • If a PDF cannot be generated, include the quote reference and visible error in support email.
  • If email sending fails, check whether Mailgun or SendGrid is configured and enabled for the project.
  • If a quote cannot be marked Invoiceable, check whether required PO details have been added.
